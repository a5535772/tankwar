# 坦克大战 — 演进计划

> **文档版本**: v1.0  
> **最后更新**: 2026-04-21  
> **维护者**: 游戏设计师  

---

## 演进总览

游戏分 **5 期** 逐步交付，每期都有清晰的"可玩里程碑"——完成任何一期，游戏都可以跑起来让人玩。

```
Sprint 1        Sprint 2        Sprint 3        Sprint 4        Sprint 5
核心机制验证     单关可玩         多关卡体验       视听打磨        扩展与发布
┌──────┐      ┌──────┐      ┌──────┐      ┌──────┐      ┌──────┐
│ 玩家 │      │ 关卡 │      │ 3+关 │      │ 音效 │      │ 菜单 │
│ 地形 │  →   │ 敌人 │  →   │ 道具 │  →   │ 特效 │  →   │ 存档 │
│ 敌人 │      │ 基地 │      │ 平衡 │      │ 打磨 │      │ 发布 │
└──────┘      └──────┘      └──────┘      └──────┘      └──────┘
  MVP验证       首个完整关卡     内容扩展        品质提升        交付完成
```

## Sprint 1: 核心机制验证（当前）

**目标**: 在 `TankTest.tscn` 中验证所有核心机制可以正常协同工作  
**时长**: 2-3 周  
**交付物**: 一个可交互的测试场景，能跑、能打、有地形、有敌人  
**完成标志**: 玩家可以在测试场景中移动、射击、看到子弹打中砖墙碎裂、看到敌人被击杀

### 1.1 战场与地形

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 战场边界 | 坦克和子弹不可超出战场范围（416×416 像素 / 13×13 格） |
| P0 | 砖墙 | 可被任何炮弹摧毁，被摧毁后从地图移除 |
| P0 | 钢墙 | 仅 Lv.3 炮弹可摧毁，其他炮弹击中后消失 |
| P1 | 水域 | 坦克不可进入，子弹可飞越 |
| P2 | 基地 | 位于底部中央，被摧毁则游戏结束 |
| P2 | 草地 | 坦克进入后对敌人不可见 |
| P3 | 冰面 | 坦克经过时有滑动惯性 |

### 1.2 敌人补全

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 敌人血量 | `take_damage()` 实际扣减血量，血量归零调用 `die()` |
| P0 | 敌人射击 | 敌人可向当前朝向发射子弹 |
| P1 | 追踪型敌人 | ChaseEnemyTank：检测到玩家后主动追踪 |
| P1 | 射击冷却 | 敌人射击有冷却时间，不会疯狂射击 |

### 1.3 玩家补全

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 玩家血量 | PlayerTank 拥有血量，被击中扣血 |
| P0 | 玩家受伤反馈 | 被击中后短暂无敌帧 + 视觉闪烁 |
| P0 | 玩家死亡 | 血量归零 → 死亡 → 可重生 |
| P1 | 玩家组注册 | 添加到 `"player_tanks"` 组（修复道具无法拾取的 bug） |

### 1.4 子弹系统补全

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 敌人子弹 | LAYER_ENEMY_BULLET 碰撞层，检测 LAYER_PLAYER |
| P0 | 子弹-地形交互 | 砖墙被击中 → 摧毁并移除；钢墙行为正确 |
| P0 | 子弹击杀者追踪 | `take_damage()` 接收正确的 attacker 参数（修复 null bug） |
| P1 | 子弹对象池 | 减少频繁实例化/销毁的性能开销 |

## Sprint 2: 单关可玩

**目标**: 完成一个完整可玩的关卡，从开始到胜利/失败有完整流程  
**时长**: 2-3 周  
**交付物**: 第 1 关完整可玩  
**完成标志**: 玩家可以从头到尾玩完一关，有胜利和失败画面

### 2.1 关卡管理器

| 功能 | 描述 |
|------|------|
| LevelManager 单例 | 管理关卡流程：开始 → 战斗 → 胜利/失败 |
| 敌人生成 | 从地图顶部预定位置按波次生成敌人 |
| 胜利条件 | 消灭所有波次的所有敌人 |
| 失败条件 | 基地被毁 或 玩家生命耗尽 |
| 关卡配置 | 使用 Resource/JSON 定义敌人波次和地图布局 |

### 2.2 基地系统

| 功能 | 描述 |
|------|------|
| 基地实体 | 底部中央，有血量，被击中会受损 |
| 基地保护墙 | 基地周围有砖墙保护 |
| 基地被毁 | 触发游戏失败 |

### 2.3 敌人生成与波次

| 功能 | 描述 |
|------|------|
| 生成点 | 地图顶部 3 个生成点 |
| 波次配置 | 定义每波敌人数量和类型 |
| 生成间隔 | 敌人依次生成，不会同时涌出 |
| 最大同屏数 | 同时存在的敌人数量上限 |

### 2.4 基础 UI

| 功能 | 描述 |
|------|------|
| 关卡开始画面 | 显示关卡编号，短暂倒计时 |
| 剩余敌人指示 | HUD 显示剩余敌人数量 |
| 玩家生命显示 | HUD 显示玩家剩余生命数 |
| 游戏结束画面 | 胜利/失败提示 + 重新开始按钮 |

## Sprint 3: 多关卡体验

**目标**: 3+ 关卡，有难度递进，有关卡选择  
**时长**: 3-4 周  
**交付物**: 完整的关卡体验链  
**完成标志**: 玩家可以连续挑战多个关卡，感受到难度提升

### 3.1 关卡内容

| 功能 | 描述 |
|------|------|
| 3-5 个关卡 | 不同地形布局、不同敌人配置 |
| 难度递进 | 后续关卡敌人更多、更强、更聪明 |
| 新敌人类型 | 巡逻型（固定路线）、炮塔型（原地旋转射击） |
| Boss 关卡 | 特殊关卡末尾出现大型坦克 Boss |

### 3.2 道具扩展

| 功能 | 描述 |
|------|------|
| 护盾道具 | 拾取后 10 秒无敌 |
| 加速道具 | 拾取后 15 秒移速 +50% |
| 生命恢复道具 | 恢复 1 点生命 |
| 道具掉落机制 | 击杀特定标记敌人掉落道具 |

### 3.3 关卡流程

| 功能 | 描述 |
|------|------|
| 关卡选择界面 | 显示已解锁/未解锁关卡 |
| 关卡解锁 | 通关前一关解锁下一关 |
| 关卡过渡 | 通关后的过渡动画 |
| 持续进度 | 通关记录保存到本地 |

## Sprint 4: 视听打磨

**目标**: 音效、特效、视觉打磨，让游戏从"能玩"变成"好玩"  
**时长**: 2-3 周  
**交付物**: 有完整音效和视觉反馈的游戏  
**完成标志**: 每个操作都有明确的音效和视觉反馈，游戏"感觉对"

### 4.1 音效系统

| 功能 | 描述 |
|------|------|
| AudioSystem 单例 | 统一管理音效播放 |
| 射击音效 | 主武器射击 + 副武器释放 |
| 爆炸音效 | 不同规模的爆炸音效 |
| 移动音效 | 坦克引擎声 |
| 道具拾取音效 | 拾取道具时的提示音 |
| BGM | 战斗背景音乐 |
| 基地警告 | 基地受攻击时的紧急音效 |

### 4.2 视觉特效

| 功能 | 描述 |
|------|------|
| 爆炸特效 | 替换占位色块，使用粒子系统或精灵动画 |
| 炮口闪光 | 射击时炮口闪光 |
| 子弹尾迹 | 高等级子弹的拖尾效果 |
| 受伤闪烁 | 坦克受伤时短暂白色闪烁 |
| 基地受损 | 基地外观随血量变化 |
| 屏幕震动 | 击杀敌人或基地受击时屏幕震动 |

### 4.3 美术替换

| 功能 | 描述 |
|------|------|
| 子弹精灵 | 替换 ColorRect 占位，使用 Retina 素材包子弹精灵 |
| 道具精灵 | 替换 ColorRect 占位，使用辨识度高的道具图标 |
| 地形瓦片美化 | 确保瓦片过渡自然 |
| HUD 美化 | 替换占位 UI 为像素风格界面 |

## Sprint 5: 扩展与发布

**目标**: 完善周边系统，准备发布  
**时长**: 2-3 周  
**交付物**: 可发布的完整游戏  
**完成标志**: 双击 exe 即可开始游戏，从主菜单到游戏结束全流程无障碍

### 5.1 完整 UI 流程

| 功能 | 描述 |
|------|------|
| 主菜单 | 开始游戏、关卡选择、设置、退出 |
| 暂停菜单 | 继续、重新开始、返回主菜单 |
| 设置界面 | 音量调节、操控设置 |
| 游戏结束界面 | 战绩统计、重新开始 |

### 5.2 存档系统

| 功能 | 描述 |
|------|------|
| SaveSystem 单例 | 保存/加载游戏进度 |
| 关卡解锁状态 | 记录已通关的最高关卡 |
| 玩家设置 | 保存音量等偏好设置 |

### 5.3 发布准备

| 功能 | 描述 |
|------|------|
| Windows 导出 | 导出为 Windows 可执行文件 |
| 性能优化 | 对象池全面应用、减少 GC 压力 |
| 输入系统迁移 | 从硬编码键位迁移到 Input Action 系统 |
| Bug 修复 | 全面测试并修复遗留问题 |

## 里程碑与依赖关系

```
Sprint 1 ──→ Sprint 2 ──→ Sprint 3 ──→ Sprint 4 ──→ Sprint 5
  地形          关卡          多关卡        音效          发布
  敌人补全      基地          道具          特效          存档
  玩家补全      波次          平衡          打磨          菜单
```

**关键依赖**:
- Sprint 2 依赖 Sprint 1 的地形和敌人射击
- Sprint 3 依赖 Sprint 2 的关卡管理器
- Sprint 4 和 Sprint 5 可以部分并行（音效制作和关卡设计可以同时进行）
- Sprint 5 的输入迁移可以在 Sprint 3 空闲期提前启动

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 地形碰撞系统复杂度超预期 | Sprint 1 延期 | P0 先做砖墙+钢墙+边界，草地和冰面降级到后续 Sprint |
| 敌人 AI 行为不自然 | 游戏体验差 | Sprint 1 先做最简行为，Sprint 3 迭代优化 |
| 子弹对象池重构影响面大 | 引入新 bug | Sprint 1 先不做对象池，Sprint 2 性能出现瓶颈时再引入 |
| 音效素材获取困难 | Sprint 4 延期 | 提前使用 BFXR/ChipTone 生成占位音效 |
