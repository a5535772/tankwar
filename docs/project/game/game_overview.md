## 1. 一句话描述

一款致敬经典 90 坦克大战、融合蝙蝠侠阿卡姆骑士坦克战斗体验的 **2D 俯视角坦克射击游戏**——你操控一辆火力可升级的坦克，在可破坏的战场上与 AI 敌人交战，保护基地，解锁副武器，每一局都是一次紧凑的战术抉择。

---

## 2. 设计支柱

这些是游戏 **不可协商** 的核心体验目标，所有设计决策以此为衡量标尺：

| 支柱 | 释义 | 玩家感受 |
|------|------|----------|
| 🎯 **一炮一决** | 每颗炮弹都有意义——打中还是打偏，摧毁掩体还是保留掩体，每个选择都有后果 | 紧张、专注 |
| 🛡️ **攻守博弈** | 保护基地 + 保护自己 vs. 主动出击消灭敌人，每一次推进都是风险/回报计算 | 策略、抉择 |
| 💥 **火力进化** | 从基础炮弹到导弹弹幕、EMP 冲击波，武器随战斗进程持续升级，战斗节奏不断加速 | 成长、爽快 |
| 🏗️ **可破坏战场** | 砖墙可击穿、钢墙需强攻、水域不可逾越——地形既是障碍也是武器 | 战术、变化 |

## 3. 核心游戏循环

### 3.1 即时循环（0-30 秒）

```
移动 → 瞄准 → 射击 → 观察结果 → 调整位置
  ↑                                          ↓
  ←←←←←←←←←← 受到威胁 / 发现敌人 ←←←←←←←←←←
```

- **行动**: WASD 四方向移动坦克，J 键发射主炮
- **反馈**: 炮弹击中敌人闪烁 → 爆炸特效；击中砖墙 → 墙壁碎裂
- **奖励**: 击杀敌人获得充能值；摧毁掩体开辟新路线

### 3.2 会话循环（5-15 分钟 / 一关）

```
关卡开始 → 消灭敌人波次 → 拾取道具升级 → 保护基地 → 关卡胜利/失败
```

- **目标**: 消灭当前关卡所有敌人
- **张力**: 基地被攻击 → 必须回防；敌人太多 → 需要合理使用副武器
- **解决**: 全灭敌人 = 胜利；基地被毁 = 失败

### 3.3 长期循环（跨关卡）

```
完成关卡 → 解锁新关卡 → 发现新敌人类型 → 调整战术 → 挑战更难关卡
```

- **进度**: 关卡解锁链
- **留存钩**: 新敌人类型逼迫玩家尝试不同武器组合

## 4. 核心机制清单

### 4.1 玩家系统

| 机制 | 描述 | 状态 |
|------|------|------|
| 四方向移动 | 经典 90 坦克式移动：上下左右，不允许斜向 | ✅ 已实现 |
| 旋转跟随 | 坦克朝向随移动方向自动调整（0°/90°/180°/270°） | ✅ 已实现 |
| 生命值 | 坦克血量系统 | ❌ 待实现 |
| 无敌/护盾 | 受伤后短暂无敌帧；护盾道具提供临时无敌 | ❌ 待实现 |
| 出生保护 | 玩家出生时短暂无敌 | ❌ 待实现 |

### 4.2 武器系统

| 机制 | 描述 | 状态 |
|------|------|------|
| 主武器·三级升级 | Lv.1 普通炮弹 → Lv.2 快速炮弹 → Lv.3 消钢炮弹 | ✅ 已实现 |
| 充能系统 | 击杀敌人掠夺充能值，用于释放副武器 | ✅ 已实现 |
| 副武器·导弹弹幕 | 消耗 50 充能，发射 8 枚追踪导弹 | ✅ 已实现 |
| 副武器·EMP 冲击波 | 消耗 40 充能，范围瘫痪敌人 3 秒 | ✅ 已实现 |
| 副武器切换 | Q 键在已解锁副武器间切换 | ✅ 已实现 |
| 敌人子弹 | 敌人坦克的射击功能 | ❌ 待实现 |

### 4.3 地形系统

| 机制 | 描述 | 状态 |
|------|------|------|
| 砖墙 | 可被任意炮弹摧毁的掩体 | ❌ 待实现 |
| 钢墙 | 仅 Lv.3 炮弹可摧毁；其他炮弹反弹 | ❌ 待实现 |
| 水域 | 坦克不可通过；子弹可飞越 | ❌ 待实现 |
| 草地 | 坦克进入后对敌人不可见 | ❌ 待实现 |
| 冰面 | 坦克经过时产生滑动惯性 | ❌ 待实现 |
| 基地 | 位于战场底部中央，被摧毁则游戏结束 | ❌ 待实现 |
| 边界墙 | 战场外围不可摧毁的围栏 | ❌ 待实现 |

### 4.4 敌人系统

| 机制 | 描述 | 状态 |
|------|------|------|
| 基础敌人·巡逻型 | 随机方向移动，不主动追踪 | ✅ 已实现 |
| 追踪型敌人 | 检测到玩家后主动靠近 | ❌ 待实现 |

## 5. 美术与音效风格

### 5.1 视觉风格

- **整体风格**: 像素艺术（Pixel Art），致敬经典 90 坦克大战视觉
- **色彩基调**: 军绿色 + 沙黄色 + 钢铁灰，搭配鲜明的 UI 高亮色
- **素材来源**: 使用 Retina 像素坦克素材包（已入库 `assets/images/Retina/`）
- **网格规范**: 32×32 像素基准网格，所有实体和地形元素对齐
- **特效**: 爆炸使用粒子系统（GPUParticles2D / CPUParticles2D）

### 5.2 音效风格（待实现）

- **BGM**: 紧凑的电子军事风，节奏随战斗强度变化
- **SFX**: 炮弹发射、爆炸、金属撞击、引擎轰鸣、道具拾取
- **反馈层次**: 每次射击 → 屏幕轻微震动 + 音效 + 炮口闪光

## 7. 技术架构概要

| 项目 | 方案 |
|------|------|
| 引擎 | Godot 4.3 (Forward+ 渲染) |
| 语言 | GDScript |
| 状态管理 | Godot State Charts 插件（已引入，未使用） |
| 事件系统 | EventBus 全局单例（信号驱动） |
| 碰撞系统 | 8 层位掩码碰撞层（CollisionLayers.gd） |
| 地图系统 | TileMapLayer + TileSet 资源 |
| 实体基类 | Tank → PlayerTank / EnemyTankBase 继承体系 |
| HUD | WeaponHUD（CanvasLayer） |
| 性能优化 | 对象池（待实现）、网格对齐 |
