# 副武器系统测试指南

## 测试前准备

### 1. 添加输入映射

在 Godot 编辑器中,打开 `项目 → 项目设置 → 输入映射`,添加以下动作:

| 动作名称 | 键位 | 说明 | 状态 |
|----------|------|------|------|
| `switch_secondary_weapon` | Q | 切换副武器 | ❌ 待添加 |

### 2. 为敌人添加分组

在测试场景中,为所有敌人坦克节点添加分组:
- 选中敌人坦克节点
- 在节点属性中找到 `Groups`
- 添加分组名称: `enemy_tanks`

**状态**: ❌ 待添加

### 3. 为地形添加分组

如果有地形节点,添加相应分组:
- 砖墙: `terrain` 和 `brick_wall`
- 钢墙: `terrain` 和 `steel_wall`
- 边界: `boundary`

**状态**: ❌ 待添加（地形系统未实现）

```

PlayerTank: HUD 已初始化
主武器: 成功发射子弹, 等级=1, 位置=..., 方向=...
充能已满!
```

### 常见问题排查

**问题: 按 K 键没有反应**
- 检查充能是否足够 (导弹需要50点, EMP需要40点)
- 查看 HUD 充能条数值

**问题: 充能不增加**
- 确认敌人有 `charge_value` 属性
- 确认击杀时触发 `EventBus.enemy_killed` 信号
- 检查敌人是否添加了 `enemy_tanks` 分组

**问题: 副武器效果不显示**
- 检查控制台是否有错误信息
- 确认 Missile.gd 和 EMPWave.gd 正确加载

**问题: HUD 不显示**
- 检查 WeaponHUD.tscn 是否正确加载
- 查看 PlayerTank 的 `_init_hud()` 是否被调用

## 性能测试

### 对象池测试

目前导弹和子弹未使用对象池,如果出现性能问题:
- 限制同屏导弹数量 (修改 MissileBarrage.gd)
- 使用对象池优化 (后续实现)

### 内存测试

运行游戏后检查内存使用:
- 每枚导弹约 5-10 KB
- 每个 EMP 效果约 2-5 KB
- 长时间运行注意内存泄漏
